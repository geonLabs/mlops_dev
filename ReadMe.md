# mlops_cvat

## Features
- CVAT webhook endpoint: `POST /cvat/webhook`
  - Handles CVAT `update:job` events
  - Re-checks all jobs in the task via CVAT API
  - Sends Gmail notification when all jobs are completed
- CVAT project list endpoint: `GET /cvat/projects`
  - Returns existing CVAT projects (`id`, `name`) for frontend dropdown
  - Requires CVAT login session
- CVAT auth endpoints
  - `POST /cvat/auth/login`
  - `GET /cvat/auth/me`
  - `POST /cvat/auth/logout`
- CVAT organizations endpoint: `GET /cvat/organizations`
  - Used to choose organization context before loading projects
- MinIO upload API group: `/upload`
  - Create upload jobs
  - Upload files directly to backend -> MinIO
  - Upload individual image file to CVAT task + MinIO in parallel: `POST /upload/image`
  - Generate presigned PUT URL
  - ZIP upload flow:
    - Create ZIP job first (`POST /upload/zip/jobs`)
    - Upload ZIP to existing job (`POST /upload/zip/{job_id}`)
    - Receive ZIP on server temp path
    - Extract ZIP and upload extracted image files to MinIO
    - Create CVAT task in selected project
    - Task name can be auto-generated by current date/time or user-provided
    - Upload to MinIO and CVAT in parallel
    - Advanced options: `segment_size`, `image_quality` (default `100`)
    - Optional cleanup for uploaded ZIP temp artifacts
  - Stream upload state by SSE

## Main Files
- `app/main.py`: single FastAPI entrypoint
- `app/routers/cvat_webhook.py`: CVAT webhook + Gmail logic
- `app/routers/upload.py`: upload/presign/SSE endpoints
- `app/core/config.py`: MinIO + CORS settings
- `app/services/minio_client.py`: MinIO client helpers
- `app/services/jobs.py`: in-memory upload job state

## Git Collaboration Rules

### Repository settings (maintainer)
1. Add members:
   - GitHub repo -> `Settings` -> `Collaborators and teams`
   - Invite members with `Write` permission (or higher)
2. Protect `main` branch:
   - GitHub repo -> `Settings` -> `Branches` (or `Rules` -> `Rulesets`)
   - Enable:
     - `Require a pull request before merging`
     - `Require approvals` (recommended: at least 1)
     - `Require conversation resolution before merging`
   - Recommended:
     - `Restrict pushes` on `main` (block direct push)
     - `Require status checks to pass` (if CI exists)

```bash
git checkout main
git pull origin main
```

```bash
git checkout -b feature/<short-topic>
```


## API Examples

### 1) Create upload job
```bash
curl -X POST http://127.0.0.1:8000/upload/jobs
```

### 2) Upload file using job_id
```bash
curl -X POST "http://127.0.0.1:8000/upload/<job_id>" \
  -F "file=@/absolute/path/file.jpg"
```

### 3) Get upload job status
```bash
curl http://127.0.0.1:8000/upload/<job_id>
```

### 4) CVAT webhook test
```bash
curl -X POST http://127.0.0.1:8000/cvat/webhook \
  -H "Content-Type: application/json" \
  -d '{"event":"update:job","job":{"task_id":1,"id":10}}'
```

### 5) ZIP upload + CVAT task create
Create ZIP job:
```bash
curl -X POST http://127.0.0.1:8000/upload/zip/jobs \
  -H "Content-Type: application/json" \
  -d '{
    "project_id": 123,
    "segment_size": 100,
    "image_quality": 100,
    "task_name": "task_20260303_235959",
    "org": "my-org-slug",
    "org_id": 1
  }'
```

Upload ZIP to the created job:
```bash
curl -X POST http://127.0.0.1:8000/upload/zip/<job_id> \
  -F "file=@/absolute/path/images.zip"
```

Watch ZIP job progress:
```bash
curl http://127.0.0.1:8000/upload/<job_id>
```

### 6) ZIP temp artifacts cleanup
```bash
curl -X DELETE http://127.0.0.1:8000/upload/zip/<job_id>
```

### 7) Individual image upload (CVAT + MinIO)
```bash
curl -X POST http://127.0.0.1:8000/upload/image \
  -F "file=@/absolute/path/image.jpg" \
  -F "project_id=123" \
  -F "segment_size=100" \
  -F "image_quality=100" \
  -F "task_name=task_20260303_235959" \
  -F "org=my-org-slug"
```
